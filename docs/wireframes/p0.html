CotaObra — Wireframes P0

5 telas críticas do Sprint 1–4 · baixa fidelidade · v1.0 · 18/05/2026

Como usar: esses wireframes comunicam estrutura (o que aparece, em que posição, em que hierarquia), não visual final. O dev frontend pode iniciar a implementação com shadcn/ui + Tailwind seguindo esse layout. Refinamento de design entra em paralelo (Figma) sem bloquear código.

1. Lista de Obras — /sites

Task CO-1-03 · Sprint 1 · P0 · Componente: tabela + filtros + ações
Notas para o frontend: shadcn/ui Table + Badge + Button. Filtros em DropdownMenu com persistência em querystring. Click na linha → /sites/:id abre SiteDetail.tsx (formulário em Sheet lateral ou rota dedicada). Empty state com CTA "Cadastre sua primeira obra".

2. Fila de Solicitações Pendentes — /quote-requests

Task CO-2-07 · Sprint 2 · P0 · Componente: cards de solicitação vinda do WhatsApp
Notas para o frontend: shadcn/ui Card com border accent para urgência. Badge "URGENTE" quando deadline < 24h. Avatar do solicitante (puxar de User.name). Origem da solicitação como ícone (WhatsApp, form web, áudio). SSE atualiza badge do menu lateral em tempo real. Click em "Revisar" abre QuoteRequestReview.tsx (modal grande ou rota dedicada).

3. Catálogo de Materiais — /materials

Task CO-1-07 · Sprint 1 · P0 · Componente: tabela paginada + modal import CSV
Notas para o frontend: Badge "CotaObra" vs "Sua construtora" indica origem (tenantId IS NULL vs do tenant). Modal de import CSV usa react-dropzone; preview das 5 primeiras linhas + validação antes de submeter; relatório de resultado com count de criados/atualizados/erros. Categoria chips com contagem em tempo real (filtro multi-select).

4. Detalhe da Cotação — /quotes/:id

Task CO-4-05 · Sprint 4 · P0 · Telona principal · Componente: cabeçalho + status fornecedores + comparativo
Notas para o frontend: Quebrar em 3 PRs (CO-4-05): (a) refactor skeleton, (b) seção fornecedores com SSE, (c) PricingComparator + ações. Tooltip do "Total Corrigido ⓘ" mostra breakdown via Radix Popover. Tabela com freeze da primeira coluna em mobile (overflow-x horizontal). Vencedor por item com badge "1º" verde; vencedor geral com fundo verde na coluna toda. Indisponível em cinza italic.

5. Revisão da Solicitação (Promover para Cotação) — /quote-requests/:id

Task CO-2-07 (parte 2) · Sprint 2 · P0 · Tela principal do comprador no fluxo dia-a-dia
Notas para o frontend: Auto-mapeamento de descrição livre → SKU usa autocomplete sobre Material.name com confiança do NLU (CO-2-02). Itens com NLU confiança < 0.7 destacados em amarelo e bloqueiam o "Disparar" até serem confirmados. Áudio original em verde claro para contexto (vem do Whisper transcript). Botão "Sugerir automaticamente" (CO-3-01) popula checkboxes ranqueados. Quando user clica em "Disparar", chama POST /api/quote-requests/:id/promote + redirect para /quotes/:newId.
